# External Integrations

**Analysis Date:** 2026-04-22

## APIs & External Services

**LLM / Chat:**
- Ollama (local, default provider) — `llama3.2` model at `http://localhost:11434`
  - SDK/Client: `langchain-ollama` 0.2.3 (`ChatOllama` in `backend/rag.py`)
  - Config: `OLLAMA_BASE_URL`, `OLLAMA_MODEL` env vars
- Azure OpenAI (cloud provider, `LLM_PROVIDER=azure_openai`) — `gpt-4o-mini` deployment
  - SDK/Client: `langchain-openai` 0.3.7 (`AzureChatOpenAI` in `backend/rag.py`)
  - Auth: `AZURE_OPENAI_API_KEY` env var OR keyless via `azure-identity` `DefaultAzureCredential` + user-assigned Managed Identity (scope `https://cognitiveservices.azure.com/.default`) on Container Apps
  - Terraform: `azurerm_cognitive_account.openai` in `infra/terraform/main.tf` (kind=OpenAI, SKU=S0, region Sweden Central)
  - Deployments: `azurerm_cognitive_deployment.chat` (gpt-4o-mini, capacity 20), `azurerm_cognitive_deployment.embed` (text-embedding-3-small, capacity 120)

**Embeddings:**
- Ollama `nomic-embed-text` — `OllamaEmbeddings` in `backend/rag.py`
- Azure OpenAI `text-embedding-3-small` — `AzureOpenAIEmbeddings` in `backend/rag.py`; same auth fallback as chat

**Avatar / Video Streaming:**
- LiveAvatar.com — photorealistic WebRTC digital twin
  - Base URL: `https://api.liveavatar.com` (hard-coded constant in `backend/config.py` to prevent SSRF)
  - Endpoints used: `POST /v1/sessions/token` (X-Api-Key), `POST /v1/sessions/start` (Bearer JWT), `POST /v1/sessions/stop`; persistent WebSocket for `agent.speak` / `agent.interrupt` / `session.keep_alive` frames
  - Transport: `httpx.AsyncClient` + `websockets` library in `backend/avatar.py`
  - Auth: `LIVEAVATAR_API_KEY` env var (empty ⇒ canvas mock mode with fake `mock-session-id`)
  - Configurable: `LIVEAVATAR_AVATAR_ID`, `LIVEAVATAR_SESSION_MODE` (default `LITE`), `LIVEAVATAR_IS_SANDBOX`, `LIVEAVATAR_VOICE` (default `en-US-AndrewMultilingualNeural`)
- LiveKit (underlying WebRTC transport, delivered by LiveAvatar) — client-side attachment via `livekit-client` 2.17.3 (`Room`, `RoomEvent`, `Track`) in `frontend/components/VideoPlayer.tsx`

**Text-to-Speech:**
- Azure Cognitive Services Speech — primary TTS when `AZURE_SPEECH_KEY` set
  - Endpoint: `https://{AZURE_SPEECH_REGION}.tts.speech.microsoft.com/cognitiveservices/v1`
  - Output format: `raw-24khz-16bit-mono-pcm`
  - SSML-based synthesis streamed via `httpx.AsyncClient.stream` in `backend/tts.py` (`_stream_tts_azure`)
  - Auth: `Ocp-Apim-Subscription-Key` header (`AZURE_SPEECH_KEY`)
  - Terraform: `azurerm_cognitive_account.speech` (kind SpeechServices, SKU F0)
- Google Text-to-Speech (gTTS 2.5.3) — local-dev fallback, no credentials required (`backend/tts.py`, `_synthesize_pcm_gtts`)
  - MP3 output decoded to PCM via `miniaudio.mp3_read_s16` at 24 kHz mono

**Speech Recognition (browser-side):**
- Web Speech API (`webkitSpeechRecognition`) — client-only in `frontend/hooks/useSpeechRecognition.ts`. Works in Chrome/Edge only.

## Data Storage

**Databases:**
- Qdrant vector DB (`COLLECTION_NAME = "cv_knowledge_base"`) — three modes in `backend/rag.py::_create_vectorstore`:
  - `QDRANT_MODE=memory` — in-process `:memory:` (POC default, not persistent)
  - `QDRANT_MODE=docker` — `QDRANT_URL` (default `http://localhost:6333`), provisioned via `docker-compose.yml` service `aicv-qdrant` on ports 6333 (REST/dashboard) and 6334 (gRPC), volume `qdrant_storage`
  - `QDRANT_MODE=cloud` — `QDRANT_CLOUD_URL` + `QDRANT_CLOUD_API_KEY` (Qdrant Cloud free tier, REST on port 6333, `prefer_grpc=false`)
  - Client: `qdrant-client` 1.12.1, wrapped by `langchain_community.vectorstores.Qdrant`

**Knowledge base source:**
- `backend/bio.txt` — plain-text CV, chunked by `RecursiveCharacterTextSplitter` (chunk size 500, overlap 50) at startup
- `backend/damir_imangulov_cv.pdf` — bundled CV PDF (generated by `backend/generate_cv.py`)

**File Storage:**
- Local filesystem only in app. Terraform state stored in Azure Blob Storage (`rg-aicv-tfstate` / container `tfstate` / key `prod.terraform.tfstate`).

**Caching:**
- In-process dict `filler_cache` in `backend/avatar.py` — pre-synthesized PCM for 8 filler phrases (warmed at startup when `ENABLE_FILLERS=true`)
- In-process `_user_sessions: dict[str, UserSession]` in `backend/avatar.py` — per-browser-tab LiveAvatar session cache (TTL 1800 s / idle eviction 120 s)
- Client-side `localStorage` key `aicv_session_id` — persisted tab-scoped UUID session ID (`frontend/lib/api.ts`)

## Authentication & Identity

**End-user auth:**
- None. Public site. Requests identified only by `X-Session-ID` header (UUID v4 validated by `UUID_RE` in `backend/config.py`).

**Service-to-service auth:**
- Azure Managed Identity (user-assigned) — `azurerm_user_assigned_identity.backend` in `infra/terraform/main.tf`; roles granted: `AcrPull` (on ACR) and `Cognitive Services User` (on Azure OpenAI). Used by `DefaultAzureCredential` in `backend/rag.py` when `AZURE_OPENAI_API_KEY` is empty.
- GitHub Actions → Azure: OIDC federated credentials (no stored SP secret). Env vars in `.github/workflows/deploy-azure.yml`: `ARM_CLIENT_ID`, `ARM_TENANT_ID`, `ARM_SUBSCRIPTION_ID`, `ARM_USE_OIDC=true`.

## Monitoring & Observability

**Error Tracking:**
- None (no Sentry, Rollbar, etc.). Structured Python logs via `logging` module (format in `backend/config.py`).

**Logs:**
- Azure Log Analytics Workspace `azurerm_log_analytics_workspace.law` (SKU PerGB2018, 30-day retention) wired to Container Apps Environment (`infra/terraform/main.tf`).
- Frontend `DevConsole` component (`frontend/components/DevConsole.tsx`) — in-browser log panel.

**Analytics:**
- Google Analytics 4 — loaded via `next/script` in `frontend/app/layout.tsx` when `NEXT_PUBLIC_GA_MEASUREMENT_ID` is set. Script tags: `https://www.googletagmanager.com/gtag/js?id=${GA_ID}` + inline `gtag('config', ...)`.
- Event helper: `frontend/lib/analytics.ts` exposes `trackEvent()` and `EVENTS` constants (`TAB_VIEW`, `CHAT_SENT`, `VIDEO_PLAY`, `VIDEO_DISCONNECT`). Events emitted from `frontend/components/VideoPlayer.tsx` (`VIDEO_PLAY`, `VIDEO_DISCONNECT`) and chat/tab components.
- GA ID is configured as GitHub Actions variable `GA_MEASUREMENT_ID` and injected at build time by the `deploy-frontend` job in `.github/workflows/deploy-azure.yml`.

## CI/CD & Deployment

**Hosting:**
- Backend: Azure Container Apps (`azurerm_container_app.backend`), consumption plan, min_replicas=0, max_replicas=3, HTTP scale rule at 10 concurrent requests, port 8000, HTTPS-only ingress.
- Frontend: Azure Static Web Apps (Free tier, `azurerm_static_web_app.frontend`) with custom domain `dimangulov.space` (`azurerm_static_web_app_custom_domain.apex`, DNS TXT token validation).
- Container images: Azure Container Registry `azurerm_container_registry.acr` (SKU Basic, admin disabled — Managed Identity only).

**CI Pipeline:**
- `.github/workflows/deploy-azure.yml` — 4-job sequence triggered on push to `main`:
  1. `terraform-infra` — `terraform init/validate/apply` with Azure Blob remote state
  2. `build-backend` — `docker build ./backend` and push `aicv-backend:{sha}` + `:latest` to ACR
  3. `deploy-backend` — `az containerapp update --image ...` rolls new revision
  4. `deploy-frontend` — `Azure/static-web-apps-deploy@v1` with `NEXT_OUTPUT=export`, `PRE_BUILD_COMMAND=npm i -g pnpm`, output dir `out`
- Setup actions used: `actions/checkout@v4`, `hashicorp/setup-terraform@v3`, `azure/login@v2`.

## Environment Configuration

**Required env vars (backend):**
- `LLM_PROVIDER` (ollama | azure_openai)
- `OLLAMA_BASE_URL`, `OLLAMA_MODEL`, `EMBED_MODEL` — when provider=ollama
- `AZURE_OPENAI_ENDPOINT`, `AZURE_OPENAI_API_VERSION`, `AZURE_OPENAI_CHAT_DEPLOYMENT`, `AZURE_OPENAI_EMBED_DEPLOYMENT` — when provider=azure_openai
- `AZURE_OPENAI_API_KEY` — optional (falls back to Managed Identity when empty)
- `AZURE_CLIENT_ID` — tells `DefaultAzureCredential` which user-assigned identity to use
- `QDRANT_MODE`, `QDRANT_URL`, `QDRANT_CLOUD_URL`, `QDRANT_CLOUD_API_KEY`
- `LIVEAVATAR_API_KEY`, `LIVEAVATAR_AVATAR_ID`, `LIVEAVATAR_SESSION_MODE`, `LIVEAVATAR_IS_SANDBOX`, `LIVEAVATAR_VOICE`, `ENABLE_FILLERS`
- `AZURE_SPEECH_KEY`, `AZURE_SPEECH_REGION`
- `ALLOWED_ORIGINS` (comma-separated CORS origins), `MAX_SESSIONS`

**Required env vars (frontend, build-time):**
- `NEXT_PUBLIC_API_URL` — backend origin
- `NEXT_PUBLIC_GA_MEASUREMENT_ID` — GA4 measurement ID
- `NEXT_OUTPUT` — `export` for static build
- `NEXT_TELEMETRY_DISABLED`

**Secrets location:**
- Azure Container Apps `secret` blocks (`infra/terraform/main.tf`): `qdrant-url`, `qdrant-api-key`, `liveavatar-api-key`, `speech-key` — mounted as env vars with `secret_name`.
- GitHub repository secrets: `AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`, `TF_STATE_STORAGE_ACCOUNT`, `QDRANT_CLOUD_URL`, `QDRANT_CLOUD_API_KEY`, `LIVEAVATAR_API_KEY`, `AZURE_STATIC_WEB_APPS_API_TOKEN`.
- Local: `backend/.env` (gitignored; existence only — contents not read).

## Webhooks & Callbacks

**Incoming:**
- None. No public webhook receiver endpoints in `backend/main.py` (routes: `/ask`, `/ask/stream`, `/speak`, `/interrupt`, `/session` GET/DELETE, `/ping`, `/health`).

**Outgoing:**
- LiveAvatar `POST /v1/sessions/stop` — fire-and-forget cleanup from `backend/avatar.py::_stop_liveavatar_session` on idle eviction or client disconnect.

## Persistent Connections

**Outbound WebSocket:**
- LiveAvatar session WebSocket (`ws_url` returned by `/v1/sessions/start`) — one per user session in `backend/avatar.py::_avatar_ws_loop`. Sends `session.keep_alive` every 3 min; handles `session.stopped`; forwards `agent.speak` (base64 PCM chunks of 48 000 bytes) and `agent.interrupt`.

**Frontend WebRTC:**
- LiveKit room joined via `room.connect(livekit_url, livekit_client_token)` — audio + video tracks auto-subscribed in `frontend/components/VideoPlayer.tsx`.
